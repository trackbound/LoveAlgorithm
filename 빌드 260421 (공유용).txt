[빌드 260421 — 이번 빌드에 뭐가 바뀌었나요?]

# 요약
- 저장/로드가 더 깔끔하고 안정적으로 동작합니다.
- 스케줄·상점 사용 흐름이 더 단순해졌습니다 (운동/공부는 바로 진행).
- 상점 구매·장바구니 표시와 알림이 말끔히 정리됐습니다.
- 설정창의 모든 슬라이더에 미리듣기 소리가 추가됐습니다.
- 프롤로그 진행과 이름 입력 흐름이 자연스럽게 바뀌었습니다.
- 로그창·튜토리얼 등 자잘한 UI 오류가 많이 고쳐졌습니다.


■ 저장 / 불러오기

· 자동 저장 슬롯 보호
  - 저장할 때 1번(자동 저장) 슬롯을 누르면
    "자동 저장 슬롯입니다. 다른 슬롯에 저장해 주세요." 안내가 뜹니다.

· 썸네일(슬롯 미리보기 그림) 개선
  - 스케줄이나 상점 화면을 열어둔 채로 저장하면, 그 화면이 썸네일에 같이 보입니다.
  - 확인 팝업이 섬네일에 같이 찍히던 문제를 고쳤습니다.

· 불러오기 후 잔상 수정
  - 스케줄 화면을 열어둔 상태에서 저장했다가 다시 불러오면
    선택지가 남아있던 문제를 고쳤습니다.

· 연속 저장 시 썸네일 갱신
  - 저장 팝업을 열어둔 상태로 여러 슬롯에 연속 저장하면
    두 번째 저장부터 썸네일이 업데이트되지 않던 문제를 고쳤습니다.


■ 스케줄

· 운동 / 공부 버튼을 수정했습니다.
  - 예전: 운동 탭 → 세부 종목 선택 → 진행
  - 지금: 운동 아이콘 누르면 바로 "운동을 진행하시겠습니까?" 팝업         
  - 알바는 이전과 동일하게 탭 + 세부 선택을 유지합니다.

· 러닝에 "끈기 +1" 효과 추가  

· 한 번의 스케줄에 하나만
  - 한 번 열린 스케줄 창에서 알바·운동·공부 중 하나만 선택할 수 있습니다.
  - 하나를 고른 뒤 다른 것을 시도하면
    "이번 스케줄에서는 이미 하나를 선택했습니다." 토스트가 뜹니다.
  - 스케줄이 다시 열리면(하루 2~3회) 다시 선택할 수 있습니다.
  - 상하차 "하루 1회" 제한은 유지했습니다.


■ 상점

· 장바구니 수량 표기 오류 수정
  - 11개 이상 담았을 때도 정확한 수량이 보입니다.
  - 두 자릿수가 되면 글자가 작아지던 문제도 고쳤습니다 (18pt 고정).

· 구매 알림 방식 변경
  - 기존: 한 번에 긴 토스트 하나
  - 변경: 여러 줄을 순서대로 보여주는 순차 토스트
    예) "합계 -1,200원" → "피로회복제 x2" → "피로 -6" → "노트북 거치대" → "지성 +2"
  - 스탯 변동(체력·피로 등)은 스케줄과 동일한 포맷("체력 +3", "피로 -5")으로 표시됩니다.
  - 하지만 지금 방식은 아이템 많이 구매시 토스트가 너무 많이 쌓이므로 다른 방식의 개선이 필요합니다.

· "COMING SOON" 분홍 Footer
  - 추가완료했습니다.

· 긴 아이템 이름 흐름 효과 개선
  - 장바구니·상세 팝업에서 아이템 이름이 길 때 좌우로 흐르는 연출이
    "한 바퀴 돌고 완전히 사라진 뒤에야 다시 나오던" 문제를 고쳤습니다.
  - 이제 글자가 끝나기 전에 다음 글자가 우측에서 바로 이어서 들어옵니다.
  - 글자 사이 간격은 아이템 이름 길이와 상관없이 항상 일정합니다.


■ 설정 창

· 모든 슬라이더에 미리듣기 소리 추가
  - 원래 BGM·SFX 슬라이더에만 있던 "미리듣기 알림음"이
    자동진행 속도 / 타이핑 속도 / 캐릭터별 음성 슬라이더에도 추가됐습니다.
  - 캐릭터별 음성 슬라이더: 슬라이더 위치(0~100%)대로 볼륨 미리듣기.
  - 자동진행·타이핑 속도 슬라이더: SFX 볼륨 그대로 미리듣기.


■ 로그 / UI

· 로그창 맨 아래까지 안 내려가던 문제 수정
  - 가끔 마지막 줄이 보이지 않던 현상을 고쳤습니다.

· 독백·주인공 말풍선 분홍 띠
  - 분홍색 그라디언트 배경 리소스가 별도로 필요합니다.

· 스케줄 튜토리얼 첫 화면
  - 튜토리얼이 처음 뜰 때 임시 안내 글자가 한 프레임 보이던 문제 수정.
  - 첫 안내 멘트가 비활성 상태로 시작해서 0.2초 동안 부드럽게 나타나도록
    변경했습니다. (두 번째 이후 멘트는 기존처럼 글자만 페이드)


■ 캐릭터 리소스

· 다은 캐릭터 이미지 7종 품질 개선


■ 스토리 / 프롤로그 흐름

· 새 게임을 시작하면 이제 바로 이름 입력 화면으로 가지 않습니다.
  - 프롤로그가 먼저 시작되고, 이야기 도중 지정된 시점에 이름 입력 창이 뜹니다.
  - 이름 입력 → 확인 팝업 → 이야기 계속 진행.
  - 입력한 이름은 게임 상태에 즉시 반영됩니다.

· 스케줄 창에 "일차"(N일차) 표시가 뜹니다 (프롤로그 1~3일차 적용).

· 희원 캐릭터의 배경을 "편의점 앞"으로 통일했습니다.


■ 시나리오 작성관련

· CSV에서 표정·배경·효과를 동시에 보여주고 싶을 때
  - 마지막 줄만 "await / click / 숫자"로 두고,
    앞의 줄들은 ">"로 연결하면 동시에 재생됩니다.
