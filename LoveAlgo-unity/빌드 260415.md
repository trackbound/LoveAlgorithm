# 빌드 260415 변경사항

## 버그 수정

- **저장/불러오기 안 되던 문제 수정** — 게임 중 저장 후 불러오기 하면 "로드 완료" 메시지만 뜨고 해당 위치로 이동하지 않던 문제를 수정했습니다.
- **데모 종료 시 스케줄 화면 잠깐 보이던 문제 수정** — 프롤로그 종료 후 스케줄 화면이 잠깐 노출되었다가 종료 팝업이 뜨던 현상을 수정했습니다. 이제 프롤로그 종료 → 검은 화면 → 데모 종료 안내 팝업 순서로 자연스럽게 진행됩니다.

## 개선

- **새 게임 시작 속도 개선** — 이름 입력 후 프롤로그 진입까지의 대기 시간을 약 1초 단축했습니다.
- **로아 가상 배경 전환 개선** — 로아의 표정에 따라 배경 오버레이가 바뀔 때, 색상이 섞이지 않고 깔끔하게 전환되도록 개선했습니다.
- **감정 배경 전환 즉시 반영** — 감정별 배경 오버레이가 바뀔 때 페이드 없이 즉시 교체되도록 변경했습니다. 표정 변화처럼 자연스럽게 바로 반영됩니다.
- **CG 표시 시 오토모드 일시정지** — CG가 화면에 뜰 때 오토모드가 자동으로 꺼지고, CG만 꽉 차게 표시됩니다. 수동으로 클릭해야 다음으로 진행됩니다. CG 종료 후에는 이전 오토모드 상태가 자동 복원됩니다.
- **스케줄 결과 토스트 순차 표시** — 행동 확정 후 효과가 여러 개일 때, 토스트 프레임은 유지한 채 효과 항목이 하나씩 페이드 전환되며 표시됩니다.
- **상점 아이템 이름 스크롤 효과** — 이름이 긴 아이템의 경우, 판매 슬롯과 상세 팝업에서 LED 전광판 스타일로 이름이 좌→우로 부드럽게 흘러갑니다. ItemCatalog에서 아이템별로 UseMarquee 체크로 개별 적용 가능합니다.
- **스케줄 튜토리얼 뼈대 제작** — 최초 스케줄 진입 시 로아가 UI 각 영역을 하나씩 설명하는 튜토리얼 오버레이를 추가했습니다. CSV 기반 17스텝 구성으로, 딤 하이라이트/로아 표정/말풍선이 클릭·스페이스바로 진행됩니다. 텍스트 분량에 따라 말풍선 크기가 자동 조절됩니다.
- **로딩 스크린 최소 시간 3초로 변경** — 로딩 화면이 너무 빠르게 사라지지 않도록 최소 표시 시간을 3초로 설정했습니다.
- **타이핑 효과음 증가** — 대사 타이핑 시 효과음 빈도를 높였습니다. (더 증가시킬까요?)
- **봄·희원 캐릭터 위치 수정** — 봄, 희원의 스탠딩 위치를 조정했습니다.
- **대사창 행간 수정** — 대사 텍스트의 줄 간격을 조정하여 가독성을 높였습니다.
- **아이템 상세 팝업 좌측정렬 및 행간 수정** — 상세 팝업의 설명 텍스트를 좌측정렬로 변경하고 행간을 조정했습니다.
- **장소 배너 이벤트명 자동 [ ] 감싸기** — Place 명령에서 이벤트명에 `[ ]`를 직접 쓰지 않아도 코드에서 자동으로 감싸줍니다. CSV에는 `새 학기 첫날 | 침대 위`처럼 쓰면 `[ 새 학기 첫날 ]` + `침대 위`로 표시됩니다.
- **스케줄 결과 팝업 글자색 및 정렬 변경** — 행동 결과 토스트의 효과 텍스트(체력+3 등) 글자색과 정렬을 조정했습니다.

## 리소스

- **다은 캐릭터 이미지 신버전 교체** — 다은 캐릭터 스프라이트 7종(Default, BrightSmile, EyeSmile, Glare, Happy, Surprise, Tearful)을 신규 일러스트로 교체했습니다.

## 참고 (CSV 작성 관련)

- 나레이션(주인공) 대사에서 히로인 표정을 바꾸려면, 해당 히로인이 화면에 있는 상태에서 `<emote=표정명/>` 태그를 사용하면 됩니다.
  - 예: `주인공: <emote=EyeSmile/>예은이 자신만만한 얼굴로 신청서를 내밀며 말했다.`
- 스케줄 화면의 일차 표시는 현재 정상 동작 (프롤로그 직후 = 1일차) 입니다.
