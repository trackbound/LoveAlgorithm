[빌드 260421 변경사항]

■ 저장/로드
- 저장 화면에서 자동 저장 슬롯(1번)을 누르면 "자동 저장 슬롯입니다. 다른 슬롯에 저장해 주세요." 안내 팝업이 뜹니다.
- 스케줄/상점 화면을 연 채로 저장하면 슬롯 썸네일에 해당 화면이 함께 보입니다.
- 이미 저장된 슬롯에 덮어쓰기 할 때 썸네일에 확인 팝업이 같이 찍히지 않도록 안전장치를 추가했습니다. (미리 캡처해 둔 깨끗한 썸네일을 사용하며, 실패 시 기존 썸네일을 유지)
- 스케줄 창을 연 상태에서 저장 후 불러오기 했을 때 선택지가 잔상으로 남는 문제를 수정했습니다. (불러오기 시 진행 중이던 선택지 UI를 강제로 정리)
- 저장 팝업을 연 상태에서 슬롯 두 개 이상 연속으로 저장하면 두 번째 이후 슬롯의 썸네일이 갱신되지 않던 문제를 수정 (매 저장 직전 다시 캡처).

■ 스케줄 / 상점
- 러닝 활동에 끈기 +1 효과가 추가됐습니다. (설명문대로 체력+끈기 모두 오릅니다)
- 상점 장바구니에서 11개 이상 담아도 정확한 수량이 표시되도록 수정했습니다.
- 상점 장바구니 수량 텍스트가 두 자릿수에서 작아지던 문제를 수정 (AutoSize 비활성, 18pt 고정).
- 상점 구매 완료 알림이 단일 토스트 → 순차 토스트로 변경되었고, 스탯 변동 표기가 스케줄과 동일한 포맷("체력 +1", "피로 -3" 등)으로 통일됐습니다.
- 상점 구매 시 효과 적용을 기획서대로 재정비:
  · 구매 = 돈 차감 + 인벤토리 적재만 수행. 스탯/피로 즉시 변동 없음.
  · Consumable(피로회복)·SessionBuff(세션 버프)는 사용 시점에 별도 처리될 예정 (별도 사용 UI 필요).
  · 선물(Gift)은 2차/3차 이벤트에서만 사용 가능 (기존 자동 사용 안 됨 — 변경 없음).
  · 동일날 중복 사용 50% 패널티 / 피로 0 클램프는 "사용" 시점에 적용됨 (기존 로직 유지).
  · 구매 완료 토스트는 합계 + 구매 항목명·수량 순으로 표시 (스탯 변동은 사용 시점에 안내).
- 운동/공부 탭은 슬롯을 거치지 않고 바로 행동 팝업이 뜨도록 변경됐습니다 (기본: 헬스장 / 독서실). 알바는 기존대로 탭+슬롯 유지. ScheduleUI 인스펙터의 체크박스로 추후 다시 탭 모드로 되돌릴 수 있습니다.
- 운동/공부 직접 행동 팝업 문구가 "운동을 진행하시겠습니까?" / "공부를 진행하시겠습니까?"로 표시됩니다.
- 스케줄 1회당 알바/운동/공부를 통합해 1개만 선택 가능 — 한 번 열린 스케줄에서 어떤 행동이든 하나를 고르면 그 세션에서는 추가 선택이 막히고 "이번 스케줄에서는 이미 하나를 선택했습니다." 토스트가 뜹니다. 스케줄이 다시 열리면(하루 2~3회) 재선택 가능. 상하차 "하루 1회" 제한은 그대로 유지.
- 설정창의 자동진행속도/타이핑속도/캐릭터별 음성 슬라이더에도 BGM·SFX 슬라이더와 동일한 프리뷰 알림음이 추가됐습니다. 캐릭터별 음성 슬라이더는 슬라이더 위치(0~1)에 비례한 볼륨으로, 자동진행/타이핑 속도 슬라이더는 SFX 볼륨 그대로 재생됩니다.
- 상점 판매 그리드를 끝까지 스크롤하면 "COMING SOON…" 분홍 띠 푸터가 가로 전체 폭으로 표시됩니다 (그리드 셀로 잡히지 않게 ignoreLayout + GridLayoutGroup 하단 패딩으로 자리 확보). 자동 설치: `LoveAlgo > Shop > Setup ComingSoon Footer`.

■ UI / 로그
- 로그 팝업에서 가끔 마지막 줄까지 스크롤되지 않던 문제를 수정 (레이아웃 강제 갱신 + 3-pass 스크롤 보정).
- 독백/주인공 행 배경용 분홍 그라디언트 스프라이트 추가: `Assets/Art/UI/Log/LogRowBg_Pink.png` (512×64, 9-slice 32/16/32/16). narrationEntryPrefab / userEntryPrefab의 행 루트에 Image로 깔면 목업처럼 분홍 띠가 표시됨.
- TMP `NarrationGlow` 머티리얼을 더 은은하게 튜닝: UnderlayDilate 0.3→1, UnderlaySoftness 0.4→1, UnderlayColor 알파 0.85→0.55. 글자 주위 분홍 헤일로가 더 넓게 퍼짐.
- 스케줄 튜토리얼이 처음 열릴 때 인스펙터의 플레이스홀더 텍스트가 1프레임 보이던 문제를 수정 (Awake에서 텍스트/알파를 즉시 0으로 초기화).

■ 캐릭터 리소스
- 다은 캐릭터 이미지 7종을 1428×3000 / maxTextureSize 4096으로 재처리 (LANCZOS 다단 축소 + UnsharpMask).

■ 스토리 / 스크립트
- CSV에 `Flow,,Day:N,>` 명령이 추가되어 스케줄 헤더에 일차가 표시됩니다 (Prologue 1~3일차 적용).
- CSV에 `Flow,,Username,>` 명령이 추가되어 프롤로그 중간에서도 이름 입력창을 띄울 수 있습니다 (페이드 → 이름 입력 → 확인 팝업 → 스토리 복귀, 입력값은 GameManager/GameState 양쪽에 즉시 반영).
- 새 게임 시작 시 더 이상 곧바로 이름 입력 화면으로 가지 않고, 프롤로그가 먼저 시작됩니다. 이름은 프롤로그 CSV의 `Flow,,Username,>` 시점에 입력합니다.
- 상세 가이드: `LoveAlgo-unity/docs/reference/csv-script-commands.md` Day 항목 참조.

■ 스토리 연출 (작성자용)
- CSV에서 표정/배경/효과를 동시에 보이게 하려면 마지막 행만 `await`/`click`/숫자로 두고, 앞 행들은 `>`로 연결하세요.
  예) `,Char,,C:Emote:Glare:Roa_PC_Negative,>` / `,BG,,...,>` / `,FX,,CharShake:C:15:0.3,await`
- 상세 가이드: `LoveAlgo-unity/docs/reference/csv-script-commands.md` "동시 연출 패턴" 항목 참조.
